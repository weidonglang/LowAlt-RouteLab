# 复试项目介绍

## 一句话介绍

LowAlt-RouteLab 是一个面向城市低空无人机任务的航迹规划与风险评估仿真系统，它把低空空域建模为 `Grid + Level + TimeSlot`，实现了从航线生成到 SkyGrid mock 冲突校验的完整演示闭环。

## 项目价值

这个项目不是普通管理系统，而是围绕低空技术与工程方向做的算法 + 工程结合项目。

它解决的问题是：

```text
给定无人机任务起点、终点、高度层和任务类型，
如何在禁飞区、障碍物、风险区和时间片占用约束下，
生成可评估、可解释、可提交空域资源系统的航线。
```

## 技术栈

- Python FastAPI：算法服务。
- Java Spring Boot：适配服务。
- Vue 3 + Vite + TypeScript：前端工作台。
- pytest / JUnit：测试。
- Docker Compose：容器化启动。

## 核心能力

- 地图建模：20x20 城市低空网格。
- 高度层建模：L60 / L90 / L120 / L150 / L180。
- 路径规划：Dijkstra / A* / Theta*。
- 风险评估：风险区、转弯、禁飞区邻近、障碍物邻近。
- 能耗估计：距离、转弯、风险网格惩罚。
- 群论增强：C8 转弯代价、D4 对称增强实验。
- TimeSlot 转换：路径转离散占用序列。
- SkyGrid 联动：Mock 冲突检查和预约提交。

## 与 SkyGrid 的关系

SkyGrid 偏空域资源平台：

- 预约审批
- 资源占用
- 冲突检测
- 通知审计

LowAlt-RouteLab 偏算法和任务规划：

- 生成航线
- 评估风险
- 估算能耗
- 转换 TimeSlot
- 调用 SkyGrid 检查和提交

二者组合后，可以形成低空任务规划闭环。

## 群论模块怎么讲

项目里没有把群论做成主线，而是作为两个轻量增强点：

1. C8 离散航向群：把 8 个移动方向编码，用模 8 运算计算转弯步数，并把转弯代价接入 A* / Dijkstra。
2. D4 二面体群：对正方形网格做 8 种旋转/反射，用来生成对称增强任务，验证算法在对称场景下的稳定性。

## 可以现场演示的流程

1. 打开前端 `http://127.0.0.1:5173`。
2. 在任务创建页选择起点、终点、高度层和算法。
3. 点击开始规划。
4. 在航线页查看网格地图和路径。
5. 在风险页查看 `riskScore`、`riskFactors`、能耗安全性。
6. 在算法对比页运行 Dijkstra / A* / Theta* 对比。
7. 在 SkyGrid 页查看 TimeSlot 占用，执行 mock 冲突检查和 mock 提交。

## 可能被问到的问题

### 为什么先做 2D 网格而不是 3D Cesium？

因为项目核心是算法闭环和工程可运行性。低空空域先用 `Grid + Level + TimeSlot` 建模，可以清晰表达空间、高度和时间占用。Cesium 3D 更适合作为后续展示层。

### 为什么 SkyGrid 用 mock？

为了保证项目独立可演示。真实 SkyGrid 未启动时，MockSkyGridClient 可以模拟冲突检查和预约提交，保证闭环不断。

### Theta* 为什么是简化版？

当前实现目标是 MVP。它基于 A* 结果做 line-of-sight 平滑，已经能展示路径平滑思想，并保证不穿越禁飞区和障碍物。

### 后续怎么扩展？

- Java 侧接 MySQL 保存任务和占用。
- RealSkyGridClient 对接真实 SkyGrid。
- 支持高度层切换和爬升代价。
- 前端增加任务历史和地图编辑。
- 部署层加入 Nginx 和持久化数据库。
